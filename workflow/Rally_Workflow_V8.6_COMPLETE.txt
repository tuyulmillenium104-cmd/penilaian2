================================================================================
                    PHASE 0: PREPARATION
================================================================================

INPUT: Campaign request from user

PROCESS:

STEP 1: FETCH CAMPAIGN DATA
```
API_CALL: GET https://app.rally.fun/api/campaigns/{campaignAddress}
EXTRACT:
  - campaign_name
  - mission_number
  - mission_title
  - required_mentions (list)
  - required_topics (list)
  - required_links (list)
  - tone_guidelines
  - gate_weights (G1, G2, G3, G4 weights)
```

STEP 2: IDENTIFY CONSTRAINTS
```
CHECK:
  - Is X Premium user? (affects link strategy)
  - Is link required in main tweet?
  - Minimum/maximum tweet count?
  - Character limits?
```

STEP 3: BUILD CAMPAIGN_DATA OBJECT
```
CAMPAIGN_DATA = {
  campaign_address: "___",
  campaign_name: "___",
  mission_number: ___,
  mission_title: "___",
  required_mentions: ["@___", "@___"],
  required_topics: ["#___", "#___"],
  required_links: ["https://___"],
  tone_guidelines: "___",
  gate_weights: {
    G1: ___,  // Content Alignment weight
    G2: ___,  // Information Accuracy weight
    G3: ___,  // Campaign Compliance weight
    G4: ___   // Originality weight
  },
  constraints: {
    is_premium: true/false,
    link_in_main_tweet: true/false,
    min_tweets: ___,
    max_tweets: ___
  }
}
```

OUTPUT: CAMPAIGN_DATA object (complete)

VALIDATION:
```
CHECKLIST:
[ ] campaign_address is not empty
[ ] campaign_name is not empty
[ ] required_mentions populated
[ ] required_topics populated
[ ] gate_weights all > 0

IF any checkbox FAILS:
  -> ERROR: Return to user, request missing data
IF all checkboxes PASS:
  -> PROCEED to Phase 1
```

================================================================================
                    PHASE 1: RESEARCH (4 WEB SEARCHES)
================================================================================

INPUT: CAMPAIGN_DATA

PROCESS:

STEP 1: EXECUTE 4 WEB SEARCHES
```
SEARCH_1: "{campaign_name} overview 2024 2025"
  -> Purpose: Basic project understanding
  -> Extract: What is it? What does it do? Who is it for?

SEARCH_2: "{campaign_name} tokenomics token utility"
  -> Purpose: Token details
  -> Extract: Token name, supply, utility, distribution

SEARCH_3: "{campaign_industry} trends comparison"
  -> Purpose: Industry context
  -> Extract: Competitors, market size, trends

SEARCH_4: "{campaign_name} statistics metrics success"
  -> Purpose: Hard numbers
  -> Extract: TVL, users, transactions, growth rates
```

STEP 2: EXTRACT FACTS
```
FOR each search result:
  EXTRACT verifiable facts:
  - Numbers (prices, amounts, percentages)
  - Dates (launch, updates, milestones)
  - Names (team, partners, investors)
  - Features (capabilities, integrations)
```

STEP 3: BUILD KNOWLEDGE_BASE
```
KNOWLEDGE_BASE = {
  project_overview: {

OUTPUT: KNOWLEDGE_BASE object (minimum 20 verified facts)

VALIDATION:
```
CHECKLIST:

================================================================================
                    PHASE 2: LEADERBOARD ANALYSIS
================================================================================

INPUT: CAMPAIGN_DATA + KNOWLEDGE_BASE

PROCESS:

STEP 1: FETCH LEADERBOARD
```
API_CALL: GET https://app.rally.fun/api/leaderboard?campaignAddress={address}&limit=10

EXTRACT for each competitor:
  - rank
  - score
  - engagement_metrics (likes, replies, reposts)
  - tweet_url (for Phase 2B web-reader)
```

STEP 2: ANALYZE SCORES
```
FOR each top 10 competitor:

  ANALYZE ENGAGEMENT:
    - Total engagement: likes + replies + reposts
    - Engagement rate: engagement / followers
    - Reply ratio: replies / total engagement

  ANALYZE POSITION:
    - Rank position
    - Score breakdown (if available)
    - Ranking trend
```

STEP 3: BUILD COMPETITOR_PATTERNS
```
COMPETITOR_PATTERNS = {

================================================================================
                    PHASE 2B: COMPETITOR DEEP ANALYSIS (web-reader)

INPUT: COMPETITOR_PATTERNS + tweet URLs

STATUS: OPTIONAL BUT HIGHLY RECOMMENDED for beating leaderboard

PROCESS:

STEP 1: USE WEB-READER TO FETCH COMPETITOR CONTENT
```
SKILL: web-reader

FOR each competitor in top_10:

  IF competitor.tweet_url exists:
    
    CALL web-reader with URL:
      - Extract full tweet content
      - Extract thread content (if thread)
      - Extract images/media descriptions
    
    STORE competitor_content = {
      url: "___",
      full_text: "___",
      tweet_count: ___,
      has_images: true/false,
      media_count: ___
    }
```

STEP 2: ANALYZE COMPETITOR CONTENT (DETAILED)
```
FOR each competitor_content:

  ANALYZE HOOK:
    - First 20 words: "___"
    - Hook type: [controversial | story | data | question | challenge | meme]
    - Hook strength: [weak | medium | strong]
    - Emotion triggered: [pain | fear | desire | belonging | curiosity | outrage]

  ANALYZE STRUCTURE:
    - Tweet count: ___
    - Character count per tweet: [___, ___, ___]
    - Technical depth: [low | medium | high]
    - Flow pattern: [linear | story | problem-solution | comparison]

  ANALYZE CTA:
    - CTA present: true/false
    - CTA location: [tweet 1 | tweet N | last tweet]
    - CTA type: [reply | like | retweet | follow | click | contest]
    - CTA format: [question | challenge | statement | incentive]
    - Reply bait quality: [none | weak | medium | strong]

  ANALYZE CONTENT ELEMENTS:
    - Statistics used: [yes/no, what stats]
    - Story element: [yes/no, what story]
    - Comparison made: [yes/no, what comparison]
    - Controversial take: [yes/no, what controversy]
    - Personal angle: [yes/no, what personal element]

  ANALYZE LANGUAGE:
    - Tone: [professional | casual | excited | urgent | neutral]
    - Reading level: [simple | moderate | complex]
    - AI markers detected: [list of AI-like phrases]
    - Template markers: [list of template-like patterns]
```

================================================================================
                    PHASE 3: GAP IDENTIFICATION
================================================================================

INPUT: COMPETITOR_PATTERNS [+ COMPETITOR_CONTENT if Phase 2B done]

PROCESS:

STEP 1: IDENTIFY UNUSED HOOK TYPES
```
ALL_HOOK_TYPES = [
  "controversial",    // Takes a contrarian position
  "story",            // Personal narrative
  "data",             // Statistics/numbers first
  "question",         // Opens with question
  "challenge",        // Challenges reader's belief
  "meme",             // Humor/meme-based
  "fear",             // Highlights risk/danger
  "desire",           // Appeals to wants/needs
  "curiosity",        // Creates knowledge gap
  "outrage"           // Triggers anger/injustice
]

IF COMPETITOR_CONTENT exists:
  USED_HOOK_TYPES = extract from COMPETITOR_CONTENT.hooks_found
ELSE:
  USED_HOOK_TYPES = []  // Unknown without web-reader

UNUSED_HOOK_TYPES = ALL_HOOK_TYPES - USED_HOOK_TYPES

RESULT: Available hook types that competitors didn't use (or underused)
```

STEP 2: IDENTIFY UNUSED EMOTIONS
```
ALL_EMOTIONS = [
  "pain",             // Loss, failure, regret

      })

    IF NOT has_loss_aversion(tweet):

      changes_made.append({
        layer: "psychology",
        type: "loss_aversion_added"

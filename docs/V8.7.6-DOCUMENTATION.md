## Architecture Overview

### Core Principles

1. **NO OVERLAP** - Each phase has unique responsibility
2. **Content Modification Only in Specific Phases** - Prevents chaos
3. **Single Version After Phase 10** - Locked selection
4. **No Changes After Phase 14B** - Content is FINAL
5. **Strict Execution** - All phases must complete
6. **Loop Protection** - Max 2x loop per phase, max 3 regenerations

### Data Flow

```
INPUT (Hook + Campaign Data)
    ↓
PHASES 0-2B (Data Gathering)
    ↓
PHASES 3-9B (Multi-Version Creation)
    ↓
PHASE 10 (LOCK - Select 1 Version)
    ↓
PHASES 11-14B (Single Version Optimization)
    ↓
PHASES 15-16 (Output - No Changes)
    ↓
OUTPUT (Final Content)
```

## Phase Details

### Phase 0: Campaign Data Fetch

**Purpose**: Fetch complete campaign data from Rally API

**API Endpoint**: `GET /campaign/{id}`

**Data Captured**:
- Campaign title, description, goal
- Rules and style guidelines
- Knowledge base
- Missions and requirements
- Reward structure

**Output**: `state.campaignData`

---

### Phase 1: Research via Web Search

**Purpose**: Gather external data for content enrichment

**Sources**:
1. Campaign website (if available)
2. Web search results (via z-ai-web-dev-sdk)
3. Knowledge base expansion

**Search Queries Generated**:
- `{topic} blockchain dispute resolution latest news`
- `smart contract disputes legal challenges crypto 2024`
- `DAO governance disputes examples blockchain`
- `decentralized justice Web3 arbitration trends`
- `AI agents blockchain autonomous commerce future`

**Output**: `state.researchData`, `state.externalFacts`

---

### Phase 2: Leaderboard Analysis

**Purpose**: Analyze current top performers

**API Endpoint**: `GET /leaderboard`

**Data Extracted**:
- Top 20 content items
- Author information
- Scores and rankings
- Content patterns

**Output**: `state.leaderboard`

---

### Phase 2B: Competitor Deep Analysis

**Purpose**: LLM-based analysis of competitor content

**Analysis Performed**:
- Writing style patterns
- Hook types used
- Emotional triggers
- Engagement elements
- Unique angles

**Output**: `state.competitorAnalysis`

### Phase 3: Gap Identification

**Purpose**: Find content opportunities

**Gap Types**:
- Topic gaps (uncovered subjects)
- Angle gaps (unique perspectives)
- Emotional gaps (missing emotions)
- Format gaps (different structures)

**Output**: `state.gaps`

### Phase 4: Strategy Definition

**Purpose**: Define content approach

**Strategy Elements**:

- Hook approach

**Output**: `state.strategy`

- Rules and style guidelines
- Knowledge base facts

- Competitor gaps

### Phase 6: Banned Items Scan

**Purpose**: Detect AI patterns and banned content

**Detection Categories**:

| Category | Examples | Severity |
|----------|----------|----------|
| AI Words | delve, leverage, paradigm | HIGH |
| AI Phrases | "picture this", "lets dive in" | HIGH |
| Smart Chars | em dash, curly quotes | MEDIUM |
| AI Patterns | "in this thread", "key takeaways" | HIGH |
| Template Hooks | "unpopular opinion", "hot take" | CRITICAL |
| Template Structures | "1/x", "thread 1/" | HIGH |

**Output**: `state.violations[]`

## SCORING SYSTEM - "200% Above Rally Maximum"

### Overview

This scoring system ensures all content exceeds Rally.fun maximum standards by 200%. The system uses weighted multi-factor analysis with statistical confidence calculations.

### Gate Structure

#### GATE UTAMA RALLY (G1-G4) - Scale 0-5

| Gate | Sub-Criteria | Weights |
|------|--------------|---------|
| **G1: Content Alignment** | messageAccuracy (0.30), terminology (0.25), brandConsistency (0.25), audienceFit (0.20) |
| **G2: Information Accuracy** | technicalAccuracy (0.30), officialConsistency (0.25), dataStatistics (0.25), properContext (0.20) |
| **G3: Campaign Compliance** | requiredHashtags (0.30), formatRequirements (0.25), styleGuidelines (0.25), disclosures (0.20) |
| **G4: Originality** | freshPerspective (0.30), personalInsights (0.25), naturalLanguage (0.25), creativeExpression (0.20) |

**Formula:**
```
Gi = (C1 × W1 + C2 × W2 + C3 × W3 + C4 × W4) / Σ(W)
where C = sub-criterion score (0-2)
```

#### GATE TAMBAHAN (G5-G6) - Scale 0-8

| Metric | Sub-Criteria |
|--------|--------------|
| **G5: Engagement Potential** | hookEffectiveness (0.25), ctaQuality (0.25), contentStructure (0.25), conversationPotential (0.25) |
| **G6: Technical Quality** | grammarSpelling (0.30), formatting (0.30), platformOptimization (0.20), mediaIntegration (0.20) |

#### PENILAIAN INTERNAL - Scale 0-10

| Metric | Weight | Calculation |
|--------|--------|-------------|
| Hook Score | 0.20 | Power pattern match + Weak opening avoidance + Required elements |
| Emotion Score | 0.20 | Trigger elements + Intensity + Body feelings + Emotion types |
| CT Score | 0.15 | Questions + Reply bait + Engagement hooks + Personal + FOMO + Controversy |
| Uniqueness | 0.15 | 10 - (Template violations × 3) - (AI patterns × 2) - (Banned words × 0.5) |
| Readability | 0.15 | Sentence length + Structure + Paragraph breaks |
| Viral Potential | 0.15 | Controversy + Emotion + Questions + Personal + Numbers + Urgency + Share-worthy |

### Overall Score Formula

```javascript
// Step 1: Normalize Gate Scores
normalizedGate = (gateScore / gateMax) × 10

// Step 2: Calculate Averages
gateAvg = Σ(normalizedGates) / 6
internalAvg = Σ(internalScores) / 6

// Step 3: Weighted Overall
overall = (0.40 × gateAvg) + (0.60 × internalAvg)

// Step 4: Statistical Confidence
mean = Σ(allScores) / count
stdDev = √(Σ(score - mean)² / count)
confidence = 1 - (stdDev / mean)
```

### Pass/Fail Criteria

```
PASS if ALL conditions met:
├── G1 Content Alignment ≥ 4/5
├── G2 Information Accuracy ≥ 4/5
├── G3 Campaign Compliance ≥ 4/5
├── G4 Originality ≥ 4/5
├── G5 Engagement Potential ≥ 8/8

├── Hook Score ≥ 9/10
├── Emotion Score ≥ 9/10
├── CT Score ≥ 9/10

FAIL → Regenerate (max 3 loops)
```
